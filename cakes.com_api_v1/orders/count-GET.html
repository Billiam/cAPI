GET api​/v1​/ orders​/count

» Cakes » Cakes.com API v1

The list of orders.

Request

Example Request 

{
  "access_token": "aaaaaaaaabbbbbbbbbbbccccccccccccddddddddd",
  "current": true
}

Request Parameters

  • access_token

    The bakery access token

    • Required: yes
    • Type: string
    • Example: "aaaaaaaaabbbbbbbbbbbccccccccccccddddddddd"
  • current

    When enabled, limits the orders count to those with a pickup date in the future.

    • Required: no
    • Type: boolean

Response

Example Response 

{
  "orders_count": {
    "accepted": 5,
    "conditional": 5,
    "cancelled": 5,
    "new": 5
  }
}

Response Parameters

  • orders_count

    Counts of orders grouped by order state

    • Required: no
    • Type: hash
    • title: orders count
    • accepted

      The count of accepted orders

      • Required: yes
      • Type: number
      • Example: 5
    • conditional

      The count of conditional orders

      • Required: yes
      • Type: number
      • Example: 5
    • cancelled

      The count of cancelled orders

      • Required: yes
      • Type: number
      • Example: 5
    • new

      The count of new orders

      • Required: yes
      • Type: number
      • Example: 5

Response Codes

Successful Response Codes

  • 200 OK

    A count of current orders

Failure Response Codes

  • 401 Unauthorized

    Access token is incorrect or missing

  • 403 Forbidden

    Permission denied to access order resource
