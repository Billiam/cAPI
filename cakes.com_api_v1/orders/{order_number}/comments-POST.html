POST api​/v1​/ orders​/{order_number}​/comments

» decopac » Cakes.com API v1

Add a comment to an order

Request

Example Request 

{
  "api_key": "aaaaaaaaabbbbbbbbbbbccccccccccccddddddddd",
  "order_number": 21,
  "comment": {
    "body": "The whipped icing has less of a sugar taste to it. Thank you for your order."
  }
}

Request Parameters

  • api_key

    The bakery API key

    • Required: yes
    • Type: string
    • Example: "aaaaaaaaabbbbbbbbbbbccccccccccccddddddddd"
  • order_number

    The order number. Parameter should replace the {order_number} in the URL.

    • Required: yes
    • Type: string
    • Example: 21
  • comment
    • Required: yes
    • body
      • Required: yes
      • Type: string
      • Example: "The whipped icing has less of a sugar taste to it. Thank you for your order."
      • min_length: 1
      • maxLength: 300

Response

Example Response 

{
  "email": "buser@cakes.com",
  "body": "We will not have this cake in stock for several days.",
  "date": "2014-02-05 18:45:57 -0600"
}

Response Parameters

An order comment

  • email

    Email address of the commenting user

    • Required: yes
    • Type: string
    • Example: "buser@cakes.com"
  • body

    Content of the order comment

    • Required: yes
    • Type: string
    • Example: "We will not have this cake in stock for several days."
  • date

    The date the comment was created

    • Required: yes
    • Type: string
    • Format: date-time
    • Example: "2014-02-05 18:45:57 -0600"

Response Codes

Successful Response Codes

  • 201 Created

    A comment resource

Failure Response Codes

  • 401 Unauthorized

    API key incorrect or missing

  • 403 Forbidden

    Permission denied to access order comments

  • 404 Not Found

    Requested order could not be located

  • 422 Unprocessable Entity

    The comment could not be saved
