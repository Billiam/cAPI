GET api​/v1​/ orders​/{order_number}​/comments

» Cakes » Cakes.com API v1

List comments for an order

Request

Example Request 

{
  "access_token": "aaaaaaaaabbbbbbbbbbbccccccccccccddddddddd",
  "order_number": 21
}

Request Parameters

  • access_token

    The bakery access token

    • Required: yes
    • Type: string
    • Example: "aaaaaaaaabbbbbbbbbbbccccccccccccddddddddd"
  • order_number

    The order number. Parameter should replace the {order_number} in the URL.

    • Required: yes
    • Type: string
    • Example: 21

Response

Example Response 

{
  "comments": [
    {
      "email": "buser@cakes.com",
      "body": "We will not have this cake in stock for several days.",
      "date": "2014-02-05 18:45:57 -0600"
    }
  ]
}

Response Parameters

  • comments

    List of order comments

    • Required: yes
    • Type: array
    • Items

      An order comment

      • Required: no
      • title: comment
      • email

        Email address of the commenting user

        • Required: yes
        • Type: string
        • Example: "buser@cakes.com"
      • body

        Content of the order comment

        • Required: yes
        • Type: string
        • Example: "We will not have this cake in stock for several days."
      • date

        The date the comment was created

        • Required: yes
        • Type: string
        • Format: date-time
        • Example: "2014-02-05 18:45:57 -0600"

Response Codes

Successful Response Codes

  • 200 OK

    List of order comments

Failure Response Codes

  • 401 Unauthorized

    Access token is incorrect or missing

  • 403 Forbidden

    Permission denied to access comments resource

  • 404 Not Found

    Requested order could not be located
