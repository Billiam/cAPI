GET api​/v1​/ orders​/{order_number}

» Cakes » Cakes.com API v1

Fetch order information by ID

Request

Example Request 

{
  "access_token": "aaaaaaaaabbbbbbbbbbbccccccccccccddddddddd",
  "order_number": 21
}

Request Parameters

  • access_token

    The bakery access token

    • Required: yes
    • Type: string
    • Example: "aaaaaaaaabbbbbbbbbbbccccccccccccddddddddd"
  • order_number

    The order number. Parameter should replace the {order_number} in the URL.

    • Required: yes
    • Type: string
    • Example: 21

Response Parameters

  • order_number

    The order number

    • Required: yes
    • Type: number
    • Example: 4100
  • order_state

    The order status

    • Required: yes
    • Type: string
    • Example: "accepted"
    • Enum: accepted, conditional, cancelled, new
  • price

    The order price, excluding tax

    • Required: yes
    • Type: number
    • Example: 41.53
  • item_number

    The cake ID being ordered.

    • Required: yes
    • Type: string
    • Example: "13992"
  • cake_name

    The name of the cake

    • Required: yes
    • Type: string
    • Example: "Dinosaur Train Express"
  • size

    Cake size

    • Required: yes
    • Type: string
    • Example: "1/2 Sheet"
  • flavor

    Cake flavor

    • Required: no
    • Type:
      • string
      • nil
    • Example: "Chocolate"
  • filling

    Cake filling flavor

    • Required: no
    • Type:
      • string
      • nil
    • Example: "Cannoli"
  • icing_flavor

    Cake icing flavor

    • Required: no
    • Type:
      • string
      • nil
    • Example: "Chocolate Fudge"
  • inscription_color

    Custom message icing color

    • Required: no
    • Type:
      • string
      • nil
    • Example: "Brown"
  • inscription

    Custom message content

    • Required: no
    • Type:
      • string
      • nil
    • Example: "Happy Birthday Andrew!"
  • notes

    Additional notes for the cake decorator

    • Required: no
    • Type:
      • string
      • nil
    • Example: "Please add a smiley face above the word happy."
  • requested_pickup

    The requested pickup date

    • Required: yes
    • Type: string
    • Format: date-time
    • Example: "2014-02-05 17:55:59 -0600"
  • order_placed

    The date that the order was placed

    • Required: yes
    • Type: string
    • Format: date-time
    • Example: "2014-02-04 14:05:00 -0600"
  • order_accepted

    The date that the order was accepted

    • Required: no
    • Type:
      • string
      • nil
    • Format: date-time
    • Example: "2014-02-04 16:18:21 -0600"
  • customer

    Customer information

    • Required: yes
    • name

      Customer name

      • Required: yes
      • Type: string
      • Example: "Andrew Aaronson"
    • phone

      Customer phone number

      • Required: yes
      • Type: string
      • Example: "612-555-1212"
    • email

      Customer email address

      • Required: yes
      • Type: string
      • Example: "aaronson@example.com"
  • image_url

    The main image URL

    • Required: no
    • Type:
      • string
      • nil
    • Example: "http://example.com/image.jpg"
  • images

    Upload image urls

    • Required: no
    • Type:
      • array
      • nil
    • Items

      An upload image

      • Required: no
      • title: image
      • image_url

        An uploaded image URL

        • Required: yes
        • Type: string
        • Example: "http://example.com/image.jpg"

    Order comments

    • Required: no
    • Type:
      • array
      • nil
    • Items

      An order comment

      • Required: no
      • title: comment
      • email

        Email address of the commenting user

        • Required: yes
        • Type: string
        • Example: "buser@cakes.com"
      • body

        Content of the order comment

        • Required: yes
        • Type: string
        • Example: "We will not have this cake in stock for several days."
      • date

        The date the comment was created

        • Required: yes
        • Type: string
        • Format: date-time
        • Example: "2014-02-05 18:45:57 -0600"

Response Codes

Successful Response Codes

  • 200 OK

    An order resource

Failure Response Codes

  • 401 Unauthorized

    Access token is incorrect or missing

  • 403 Forbidden

    Permission denied to access order resource

  • 404 Not Found

    Requested order could not be located
